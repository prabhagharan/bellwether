## 1. Goal

A minimal but complete **Next.js dashboard** over the bellwether REST API ‚Äî the single-user operator UI for monitoring high-influence figures, curating sources + golden labels, and configuring alerts.

**In scope ‚Äî seven pages** (all behind login): **Live Feed** (SSE alerts + recent signals), **Watchlist** (figures/sources + trigger discovery), **Review-and-correct** (Track-A golden labels), **Discovery review** (confirm/reject proposed sources), **Alerts** (rule CRUD), **Impact/Leaderboard** (Track-B analytics), **Login**. Plus a typed API client generated from the backend's OpenAPI, centralized auth, and Vitest/RTL tests for the non-visual logic.

**Out of scope (deferred with intent):** the eval/version panel (no API ‚Äî Plan 5 made optimize/programs/eval a CLI); optimistic UI; multi-user; deployment/hosting; dark mode; real-time push for anything but alerts (discovery status is poll/refresh).

## 2. Hard constraints

- **Client-rendered SPA.** Internal single-user tool, no SSR/SEO needs; pages are client components. No server-side secrets in the frontend.
- **The generated typed client is the single API boundary.** Every call goes through `openapi-fetch` typed against the backend's `/openapi.json` ‚Äî an API drift (renamed route/field) becomes a TypeScript build error. No ad-hoc `fetch` to the API.
- **One token, one attach point, one 401 path.** JWT in `localStorage`; a client middleware attaches `Authorization: Bearer` to every request and, on any 401, clears the token + redirects to `/login`. The SSE `/stream` gets the token via `?token=` (the backend's `resolve_token`).
- **Async discovery is surfaced, never blocked on.** Adding a figure returns `discovery_status="pending"`; the UI shows the status and revalidates so sources appear once the worker runs ‚Äî no spinner that waits on the worker.
- **Owner-scoping is the backend's job.** The UI never sends an owner id; every endpoint is already scoped by the authenticated user.
- **Lean dependency set.** Next, React, TypeScript, Tailwind, `openapi-fetch`/`openapi-typescript`, SWR, Vitest + React Testing Library. Nothing else.
- **Testing:** Vitest + RTL for the non-visual logic (client middleware, token store, `useAlertStream`, forms/components, empty/error states); the pages are verified live against the running backend (the backend's "unit-test the logic, live-verify the integration" discipline).

## 4. Auth & the typed API client

- **`auth/token.ts`** ‚Äî `getToken()/setToken()/clearToken()` over `localStorage["bw_token"]`.
- **`api/client.ts`** ‚Äî `openapi-fetch` client with a middleware: `onRequest` adds `Authorization: Bearer ${getToken()}` (when present); `onResponse` ‚Äî if `res.status === 401`, `clearToken()` + `window.location.assign("/login")`. So pages call `client.GET("/figures")` etc. and never handle auth.
- **Login (`/login`, public)** ‚Äî form ‚Üí `POST /auth/token` as `application/x-www-form-urlencoded` (`username`, `password`) ‚Üí `{access_token}` ‚Üí `setToken` ‚Üí redirect `/`. Wrong creds ‚Üí inline error.
- **Auth guard (`auth/guard.tsx`)** ‚Äî wraps the shell: if `!getToken()` redirect to `/login`; the login page redirects to `/` when a token exists. The token is trusted until a real 401 (handled by the middleware) ‚Äî no extra validation round-trip (a `GET /me` is available if we want a startup check; v1 skips it).
- **SSE auth** ‚Äî `useAlertStream` opens `EventSource(`${API_BASE}/stream?token=${getToken()}`)`; on `onerror` after an auth failure the browser can't read the status, so the hook treats repeated failures as disconnected and the app's normal 401 path (from any concurrent fetch) handles logout.

## 5. Pages

A shared **AppShell** (client layout): top nav ‚Äî **Feed ¬∑ Watchlist ¬∑ Review ¬∑ Discovery ¬∑ Alerts ¬∑ Impact ¬∑ Logout** ‚Äî + the auth guard. Seven routes:

- **`/login`** ‚Äî ¬ß4.
- **`/` ‚Äî Live Feed:** *Live alerts* panel (`useAlertStream` ‚Üí prepend each `event: alert` payload ‚Äî figure, direction/magnitude/confidence, message ‚Äî with a connected indicator) + *Recent signals* table (`GET /signals`, SWR; filters figure_id/direction/min_confidence).
- **`/watchlist`:** figures list (`GET /figures`); **add figure** (`POST /figures {name, type, discover:true}`); per-figure sources (`GET /figures/{id}/sources`) with `status`/`verified`/`discovery_confidence` badges; **re-run discovery** (`POST /figures/{id}/discover`); delete figure (`DELETE /figures/{id}`). Shows each figure's `discovery_status`.
- **`/review` ‚Äî Review-and-correct (golden labels):** `GET /review/queue?module=extract` ‚Üí per item show the statement text + the model's extraction; actions **Confirm** (`POST /review/{id} {is_relevant:true}`), **Correct** (edit direction/magnitude/entities/evidence_quote ‚Üí `{is_relevant:true, extraction:{‚Ä¶}}`; a non-verbatim evidence quote ‚Üí 422 shown inline), **Reject** (`{is_relevant:false}`). Item leaves the queue after (revalidate).
- **`/discovery` ‚Äî Discovery review:** `GET /discovery/queue` ‚Üí per `pending_review` source show figure name, connector type + config, `discovery_confidence`, and `discovery_meta` (which signals fired); actions **Confirm** (`POST /discovery/{source_id} {decision:"confirm"}`) / **Reject** (`{decision:"reject"}`). Item leaves the queue after.
- **`/alerts` ‚Äî Alert config:** `GET /alert_rules`; **create** with a **ConditionBuilder** (min_confidence, min_magnitude, directions, figure_ids) + webhook URL + enabled (`POST /alert_rules`); toggle enabled (`PATCH /alert_rules/{id}`); delete (`DELETE /alert_rules/{id}`).
- **`/impact` ‚Äî Impact & Leaderboard:** ranked leaderboard table (`GET /leaderboard` ‚Äî figure, n, avg_pct_move, avg_abs_pct_move, directional_hit_rate) + a filterable measured-impacts list (`GET /impacts` ‚Äî filters figure_id/symbol/window).

Each page is a focused client component over the typed client; the shell owns nav + auth. No page reads another's internals.

## 6. Data flow, loading/error, async edges

- **Reads (SWR):** each list is `useSWR(key, fetcher)` where the fetcher calls the typed client; SWR provides caching, revalidate-on-focus, `isLoading`/`error`. After a write, `mutate(key)`.
- **Live alerts (`useAlertStream`):** opens `EventSource` on mount, parses `event: alert` ‚Üí prepends to a capped list (~100), tracks connected/disconnected, closes on unmount; `EventSource` auto-reconnects transient drops.
- **Writes:** `POST/PATCH/DELETE` via the typed client ‚Üí `mutate` the relevant key (simple revalidate-after; no optimistic UI in v1).
- **Loading/error/empty UI:** every panel renders a loading state, an inline error on a failed read/write (401 handled globally), and friendly empty states ("No figures yet ‚Äî add one"; "No alerts yet ‚Äî they appear as signals match your rules"; "Review queue empty").
- **Async discovery:** `/watchlist` shows `discovery_status` (pending ‚Üí running ‚Üí done/failed) and auto-revalidates `/figures` + the sources list so sources appear after the worker runs ‚Äî never a blocking spinner.

## 7. Testing (Vitest + RTL for logic; live for pages)

- **Unit:** `auth/token.ts`; the `client.ts` middleware (attaches Bearer; on 401 clears + redirects ‚Äî mock fetch/localStorage/`location`).
- **Hook:** `useAlertStream` (mock `EventSource`; emit an alert ‚Üí assert prepend + connected state; unmount ‚Üí close).
- **Components:** login form (submit ‚Üí token stored ‚Üí redirect); `ConditionBuilder` (emits the correct `condition` object, incl. omitting empty fields); a list renders rows given data + its empty/error states; the Correct form surfaces a 422 verbatim error.
- **Compile-time net:** the generated typed client + `tsc` ‚Äî an API mismatch fails the build.
- **Manual (pre-done):** the full flow against the running backend ‚Äî login ‚Üí feed streams a real alert ‚Üí add a figure ‚Üí discovery fills sources ‚Üí review-and-correct an extraction ‚Üí confirm a discovered source ‚Üí create a rule ‚Üí watch it fire ‚Üí check the leaderboard.
- **Runner:** Vitest (jsdom) + RTL, `npm test`.

## 8. Dev / build / run

- `npm run dev` ‚Üí Next dev server on `:3000`; the backend runs separately (`uvicorn` + the workers). `NEXT_PUBLIC_API_BASE` points at the API.
- **CORS:** add `http://localhost:3000` to the backend's `cors_origins` (a `.env` line ‚Äî supported since 7a).
- `npm run gen:api` regenerates `src/api/schema.ts` from `/openapi.json` (run whenever the API changes).
- `npm run build` for a production build; `.gitignore` for `node_modules/` + `.next/`. Deploy/hosting deferred (runs locally / wherever Node runs).

## 9. Deferred with intent

- Optimistic UI; multi-user; auth refresh tokens; deployment/hosting; dark mode.
